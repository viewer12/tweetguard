Prompt 编辑

TweetGuard 发给你 AI 的所有提示词都列在这里，可看可改可还原。默认 prompt 经过 prompt-injection 防御与多语言优化，建议先观察后再改。

每条灰区推文都用这个 prompt 判断是否为 spam，返回 JSON 决策 + 可选的泛化签名。每次调用 ~700 input tokens。
使用默认 Prompt 0 字符
编写 Prompt 的建议
  • 保留 SECURITY 段 — 防止 bot 在推文里注入指令操纵你的 AI
  • 保留输出 JSON schema — 否则下游解析会失败（缓存 / 学习规则）
  • 分类 prompt 控制"什么算 spam"；复审 prompt 控制"AI 学错了规则之后如何反思"
  • 新增类别时同步在「规则」tab 添加对应模块开关，否则统计图表不显示
  • 建议把你最近遇到的特定 spam 模式举例放进 prompt 末尾（few-shot），命中率会显著提升

关于 TweetGuard

隐私承诺

  • TweetGuard 团队 不收集任何数据，没有任何后端服务器
  • 所有配置、缓存、关注列表都只存在你的浏览器本地
  • 启用 AI 后，部分推文会发送到 你自己配置的 AI 提供商，TweetGuard 不参与中转
  • 想绝对隐私？选 Ollama 本地模式 — 推文永远不离开你的设备

它是怎么工作的

  1. L0 规则：17 项快速信号 + 10 条硬规则，毫秒内决断 ~80% 显性 spam
  2. 缓存：对每个账号的判定缓存到本地，下次遇到瞬时应用（无 AI 调用）
  3. AI：仅对 L0 灰区（评分 20-70）的推文调用一次 AI 复核
  4. 面包屑：被识别的推文平滑收起为 40px 细条，平均 350ms 过渡，可一键展开
